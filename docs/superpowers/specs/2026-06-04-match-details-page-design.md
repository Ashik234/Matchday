# Match Details Page — Design Spec

**Date:** 2026-06-04
**Branch:** `feat/matchday-impl`
**Author:** assistant

## Goal

A dedicated `/match/:slug` page reachable from any match card on a
country profile, with six tabs (Overview, Head-to-Head, Recent Form,
Top Scorers, Squad Comparison, Previous Meetings) driven by
`openfootball/worldcup.json` (current WC) plus a one-time scrape of
the [`jfjelstul/worldcup`](https://github.com/jfjelstul/worldcup)
public-domain CSV database (all historical WCs through 2022).

## Non-Goals

- Statistics tab — xG, possession, pass accuracy, shots on target,
  tackles, corners. No free source.
- Weather. No free source.
- Formation diagrams on a pitch — no formation data.
- Market value — no free source.
- VAR events (the data isn't cleanly extractable from sources we use).
- Player profile sub-pages (clicking a player name).
- Live in-match event feeds for in-progress 2026 games.

## Constraints

- Same data sources as the rest of the app: free, no API key.
- Drop-in route addition — existing pages untouched.
- Respect `useReducedMotion`. No new dependencies beyond what is
  already installed.
- Bundle the historical data as a single trimmed JSON
  (~2 MB) at `public/data/match-history.json`, generated by a
  one-time `pnpm scrape:matches` script, committed to git like the
  FIFA-ranking and squads data.

## Routing

- Path: `/match/:slug`
- Slug shape: `<home-slug>-vs-<away-slug>-<YYYY-MM-DD>`
  (e.g. `brazil-vs-morocco-2026-06-14`). Date suffix prevents
  collisions when the same two teams meet twice in a tournament.
- Helpers in `src/utils/matchSlug.ts`:
  - `toMatchSlug(match: Match): string`
  - `findMatchBySlug(matches: Match[], slug: string): Match | undefined`
- 404 fallback: "Match not found" + button back to `/`.

## Data shapes

### `src/data/types/matchHistory.ts`

```ts
export type HistoricalEventType =
  | 'goal'
  | 'own-goal'
  | 'penalty'
  | 'yellow'
  | 'red'
  | 'sub';

export type HistoricalEvent = {
  minute: number;
  type: HistoricalEventType;
  teamCode: string;
  player: string;
  detail?: string;       // e.g. assist player, sub-out, "(pen)"
};

export type HistoricalMatch = {
  id: string;            // jfjelstul match_id
  tournament: string;    // e.g. "2022 FIFA World Cup"
  date: string;          // ISO YYYY-MM-DD
  stage: string;         // "Group A", "Quarter-final", "Final"
  home: { code: string; name: string; score: number };
  away: { code: string; name: string; score: number };
  stadium?: string;
  attendance?: number;
  referee?: string;
  events: HistoricalEvent[];
};

export type MatchHistoryPayload = {
  scrapedAt: string;
  matches: HistoricalMatch[];
};
```

## Scraper

`scripts/scrape-match-history.mjs`:

- Source CSVs (raw GitHub URLs):
  - `data-raw/matches.csv`
  - `data-raw/goals.csv`
  - `data-raw/bookings.csv`
  - `data-raw/substitutions.csv`
  - `data-raw/tournaments.csv`
  - `data-raw/teams.csv`
- Fetch via `node:fetch`, parse with a small hand-written CSV parser
  (RFC-4180-aware, the CSVs are clean — no need for a dep).
- Filter to men's WCs only (`tournament_name LIKE "%FIFA World Cup%"`,
  `women == FALSE`).
- For each match: join goals + bookings + substitutions by `match_id`,
  emit one `HistoricalMatch` with events sorted by `minute`.
- Map jfjelstul team names to the codes used in our app by exact-name
  comparison against the openfootball teams payload. Unknown teams
  keep their FIFA code from jfjelstul if present.
- Trim aggressively: only keep the fields listed in `HistoricalMatch`.
- Output: `public/data/match-history.json`. Target size ≤ 2.5 MB
  (pretty-printed; gzipped over the wire is ~400 KB).
- Add npm script: `"scrape:matches": "node scripts/scrape-match-history.mjs"`.
- On scrape failure: exit non-zero, leave any prior JSON intact.

## Data layer

### `src/data/api/matchHistory.ts`

```ts
const SOURCE = '/data/match-history.json';

export const matchHistoryApi = {
  all: async (_: void, signal: AbortSignal): Promise<MatchHistoryPayload> => {
    const res = await fetch(SOURCE, { signal });
    if (!res.ok) throw new Error(`match-history.json ${res.status}`);
    return res.json() as Promise<MatchHistoryPayload>;
  },
};
```

### `src/data/queries/useMatchHistory.ts`

```ts
const TTL_MS = 30 * 24 * 60 * 60 * 1000; // 30 days — bundled data
export function useMatchHistory() {
  return useQuery<MatchHistoryPayload, Error>({
    queryKey: ['match-history'],
    queryFn: ({ signal }) => matchHistoryApi.all(undefined, signal!),
    staleTime: TTL_MS,
  });
}
```

### `src/data/queries/useMatchProfile.ts`

Composite hook. Given a slug, resolves to:

```ts
type MatchProfile = {
  match?: Match;             // current-WC fixture
  h2h: HistoricalMatch[];    // every historical meeting between the two sides
  squadHome: Player[];
  squadAway: Player[];
  recentHome: Match[];       // last 5 finished matches of home team (any WC)
  recentAway: Match[];
  topScorersHome: Scorer[];
  topScorersAway: Scorer[];
  isLoading: boolean;
  notFound: boolean;
};

type Scorer = { name: string; goals: number; lastTournament: string };
```

Internally calls `useAllMatches`, `useMatchHistory`, `useSquad` (for
both teamCodes), then performs in-memory filters / aggregations
inside `useMemo`. The H2H + scorers logic is pure; expose those
helpers as testable pure functions in `useMatchProfile.ts` (not
exported as hooks).

## Hero — `MatchHero.tsx`

Layout (desktop):

```
[ flag XL ]    [ stage · matchday ]            [ flag XL ]
 BRAZIL                Brazil vs Morocco              MOROCCO
 #1 FIFA          Sat 14 Jun · MetLife Stadium       #14 FIFA
                       Referee · Attendance
                        [countdown or FT]
```

- Flags use `<FlagRipple>` size `xl` (same hover-ripple as team page).
- Countdown via existing `Countdown` component if `kickoff > now`,
  otherwise show score / "FT" / "LIVE".
- Referee + attendance shown only when present in any matched
  historical record (for upcoming matches, both will be empty —
  hide that row entirely).
- Background: blue stadium radial (reuse the gradient already used
  on `StadiumBackdrop`).

## Tabs — `MatchTabs.tsx`

Sticky below hero. Same layoutId underline pattern as `TeamTabs`.

Tab keys: `overview | h2h | form | scorers | squad | history`.

Hash-driven URL state (`#h2h`, `#form`, etc) so deep-link works.

## Tabs — content

### Overview (`MatchOverviewTab.tsx`)

- Competition: "FIFA World Cup 2026"
- Group / stage: from `Match.stage` + `Match.group`
- Venue: `Match.stadium.name` (city if available)
- Matchday: from `Match.stage` parsed (e.g. "Group A · Matchday 3")
- Kickoff: formatted full date + time
- Pre-match comparison strip: two cards side-by-side showing rank,
  GF/GA in current WC so far, last-5 form pills.

### Head-to-Head (`HeadToHeadTab.tsx`)

- Totals row: `Brazil 4 · 1 Draws · Morocco 0`.
- `H2HSummary` bars (one row per outcome).
- Goal totals: `Brazil 12 · Morocco 5` derived from `h2h.events`
  filtered by `type === 'goal' | 'penalty' | 'own-goal'`.
- Empty state: "No previous meetings on record."

### Recent Form (`RecentFormTab.tsx`)

- Two columns, last 5 finished matches each.
- Sources: openfootball current-WC finished matches first, then
  historical from `match-history.json` to fill to five.
- Each pill: opponent flag + score + W/D/L colour, click → opens that
  match's page (only if it has a slug-resolvable id; otherwise pill
  is non-interactive).

### Top Scorers (`TopScorersTab.tsx`)

- Top 5 all-time WC goalscorers for each team.
- Source: aggregate `match-history.json` goal events grouped by player
  per team.
- Each `ScorerCard`: name, total goals, last tournament played in.
- "Active in 2026" badge if the player's name appears in the team's
  `squads.json` roster (case-insensitive last-name match).

### Squad Comparison (`SquadCompareTab.tsx`)

- Two columns side-by-side.
- Each shows the team's squad sorted by jersey, grouped by position
  (GK first, then DF/MF/FW).
- Header row per column: avg age + total intl goals.
- No formation diagram. No market value.

### Previous Meetings (`PreviousMeetingsTab.tsx`)

- Reverse-chronological list of every historical meeting.
- Each row collapsed by default: date · tournament · stage · score.
- Click row → expands an `EventTimeline` showing every goal / card /
  substitution from `match.events`.
- Empty state: "No previous meetings on record."

## Components

- `H2HSummary.tsx` — three horizontal bars (home wins, draws, away
  wins) with counts. Animates width on viewport enter.
- `FormStrip.tsx` — row of W/D/L pills with opponent flags.
- `EventTimeline.tsx` — vertical timeline of `HistoricalEvent[]`
  sorted by minute. Icons per type. Player + detail.
- `ScorerCard.tsx` — name + goals + last-tournament + active badge.
- `SquadColumn.tsx` — one team's positional squad list, used in
  Squad Comparison tab.

## Match-card click-through

`src/pages/TeamPage/components/MatchTimeline.tsx` and the upcoming
carousel in `src/pages/TeamPage/tabs/MatchesTab.tsx` currently render
clickable cards that expand inline. Change: wrap each card in
`<Link to={`/match/${toMatchSlug(m)}`}>`. Drop the existing
expand-inline behaviour (deep page replaces it).

## Loading / empty / error

- Skeleton hero + skeleton tab during load (same `Skeleton` primitive).
- If `match-history.json` 404s: H2H, Top Scorers, Previous Meetings
  tabs render a "Historical data not yet scraped. Run
  `pnpm scrape:matches`." note. Other tabs still work.
- 404 slug → centred "Match not found" card with a back-to-home link.

## Accessibility

- Tabs use `role="tablist"` / `role="tab"` / `aria-selected`.
- Each event row in `EventTimeline` has an `aria-label` summarising
  it ("Goal · 12' · Vinicius Jr.").
- Match-card links use full team-vs-team text, not icon-only.
- Reduced motion: disables bar animations + count-ups + tab fade.

## Visual style

- Same blue stadium gradient + gold accents.
- Glassmorphic cards: `bg-bg-elev1/40 backdrop-blur-md border-white/8`.
- Tab underline = gold (same `layoutId` pattern as `TeamTabs`).
- W/D/L colours match `RecentForm` on the team page
  (green / gold / red).
